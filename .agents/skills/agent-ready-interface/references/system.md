# Agent Ready interface patterns

## Direction

Agent Ready uses a documentation-first product feel for Convex developers. The UI should feel like a readable project workspace: clear, technical, warm, and easy for new developers to follow.

## Navigation

Keep `Install guide` pointed at the canonical repo README install section:

`https://github.com/waynesutton/agent-ready-component#install`

Keep `Docs` pointed at the in-demo docs route:

`/docs`

Place `Docs` before `Resources` in demo navigation so new users see product docs before reference links. Keep `https://diffs.com/docs` as a supporting reference link for Diffs by Pierre, not as the Agent Ready docs destination.

## Resources

When showing resources, put `/docs` first as `Agent Ready docs`, then `https://diffs.com/docs` as `Diffs by Pierre`, then Convex docs, component docs, auth docs, and agent discovery specs.

## Docs structure

Use `README.md#install` as the canonical install guide. Keep `docs/install.md` and `docs/install.html` as lightweight entry points back to the README.
